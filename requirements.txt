# This project's own package (src/climate_change_on_sea_urchins/), which
# provides both the analysis pipeline and the dashboard (app.py imports it).

-e .

# Core scientific stack
# pandas pinned below 3.0: that major bump (mandatory copy-on-write, many
# API changes) is untested against this app and was installed unpinned on
# Streamlit Cloud (pandas 3.0.3) while local dev/testing used 2.x — a
# plausible contributor to the app hanging/blanking in production but not
# reproducing reliably locally. statsmodels has also had documented pandas
# 3.0 compatibility bugs. Revisit the pin once pandas 3.x has been
# deliberately tested against the ARIMA/SARIMAX code paths.
pandas>=2.0,<3.0
# numpy/scipy upper-bounded for the same reason as pandas above: unpinned,
# Streamlit Cloud resolved numpy 2.5.1 / scipy 1.18.0 (barely-released at
# the time) while local dev/testing used numpy 1.26.4 / scipy 1.15.3
# throughout. Production segfaulted repeatedly under real use (heavy
# statsmodels/numpy/scipy calls -- SARIMAX, correlations, MHW analysis);
# pinning Python itself via runtime.txt/.python-version did NOT fix it
# (confirmed: still crashed on Python 3.12.13), which points at these
# specific numpy/scipy releases rather than the Python version. Bounded to
# the exact range already proven stable through this whole project's local
# testing. Revisit once numpy 2.5.x/scipy 1.18.x have more production
# mileage elsewhere.
numpy>=1.24,<2.0
scipy>=1.11,<1.16
statsmodels>=0.14
scikit-learn>=1.3

# MHW->EC50 robustness battery (mhw_robustness.py): Convergent Cross Mapping
# and wavelet coherence, two of five independent checks run alongside the
# primary CCF/Granger/SEA/DLNM analyses.
skccm==0.2.dev0
pycwt==0.4.0b0

# Visualization & dashboard
plotly>=5.18
# Upper-bounded below the Tornado->Uvicorn internal server migration
# (happened between 1.56 and 1.57): unpinned, Streamlit Cloud silently
# picks up whatever is newest on every redeploy, and production started
# failing to handle a second script rerun (tab switch, button click -- the
# very first interaction after load produces a blank page with zero
# server-side trace) right around when it drifted onto a 1.57+ build.
# Revisit once that migration has had time to mature upstream.
streamlit>=1.35,<1.57
Pillow>=10.0

# Marine Heatwave detection: own implementation in mhw_detection.py, no
